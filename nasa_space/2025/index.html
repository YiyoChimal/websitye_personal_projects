NASA Space Apps Challenge 2025 – Cloud Computing for Cleaner Skies

Objective

Design and develop an Internet of Things (IoT) device for environmental monitoring capable of collecting local air quality data and transmitting it to the cloud for remote analysis and visualization.

Technologies Used

• KiCad – Electronic schematic capture and PCB design.

• ESP32-WROOM-32UE – Main embedded controller with integrated Wi-Fi.

• Arduino IDE – Embedded firmware development.

• Wi-Fi – Wireless communication for cloud connectivity.

• Cloud IoT Platform – Remote data storage and visualization.

My Contributions

• Designed the custom PCB using KiCad.

• Programmed the ESP32-WROOM-32UE using the Arduino IDE.

• Implemented Wi-Fi communication for periodic cloud data transmission.

• Integrated the sensing, embedded control, and cloud communication subsystems into a compact IoT device.

Technical Highlights

• Custom-designed PCB.

• ESP32-based IoT architecture.

• Wi-Fi cloud connectivity.

• Real-time monitoring of ozone, methane, and particulate matter (PM1.0, PM2.5, and PM10).

• Cloud-based environmental data storage and visualization.

• Mobile application for remote monitoring.

• Compact and low-power embedded system.

Results

Successfully developed a compact IoT air quality monitoring device capable of detecting ozone, methane, and particulate matter (PM1.0, PM2.5, and PM10). The system periodically transmits environmental data to the cloud via Wi-Fi, enabling remote analysis and visualization through a mobile application. This project strengthened practical experience in cloud-connected IoT architectures, embedded systems, wireless communication, and environmental monitoring.
